Manual Testing Guide

Step-by-step checklist to verify the Refund, Cancellation, and User Wallet flows

Quick Links

1. Setup Requirements

  • Have 3 accounts ready: User, Tour Guide, Admin.
  • Have at least 2 paid Tour Bookings (to test Refund & Alternative).
  • Have an Airline Booking (Economy/Premium/Business).
  • Have a Hotel Booking.

2. Tour Guide Cancels a Tour

Page: /tourguide/emergency-cancel.html

API: POST /api/tourguide/urgent-requests/submit

Action: Login as Guide -> Enter Tour ID and Reason -> Submit

Verify: Success message appears. Affected bookings become PendingUserDecision.

3. User Chooses Refund

Page: /user/tour-cancellations.html

API: GET /api/users/tour-cancellations
POST /api/users/tour-cancellations/{id}/refund

Action: Login as User -> View affected booking -> Click "Take Full Refund"

Verify: Booking disappears from pending list. Wallet balance increases by 100% of price.

4. User Checks Wallet

Page: /user/wallet.html

API: GET /api/users/wallet

Action: Open Wallet page

Verify: Balance includes refund. A "Refund" transaction appears in the history.

5. User Chooses Alternative Tour

Page: /user/tour-cancellations.html

API: GET .../alternatives
POST .../choose-alternative

Action: Open pending booking -> View Alternatives -> Select an alternative

Verify: Modal closes, booking removed from pending list. Wallet does NOT increase.

6. Admin Reviews Guide Reason

Page: /admin/tour-guide-cancellations.html

API: GET /api/admin/tour-guide-cancellations/pending-review

Action: Login Admin -> View review list -> Accept/Reject reason

Verify: Review disappears from pending list.

7. Duplicate Refund Check

Action: Attempt to manually fire the POST refund endpoint twice using Postman or console.

Verify: The second request returns "Refund has already been processed" and wallet balance only increases once.

8. Airline Cancellation Test

Action: Cancel an airline booking.

Verify: Economy/Premium = $0 refund. Business = 90% refund added to Wallet.

9. Hotel Cancellation Test

Action: Cancel a hotel booking.

Verify: Refund matches policy. Money goes to Wallet.

10. Tour User Cancellation (Normal)

Action: User cancels a normal tour booking (not guide initiated).

Verify: Before 24h = 100% refund. Within 24h = Error/Rejected.
